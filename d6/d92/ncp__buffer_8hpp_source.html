239  /*
240  * NcpFrameBuffer Implementation
241  * -----------------------------
242  *
243  * NcpFrameBuffer internally stores a frame as a sequence of data segments. The data segments are stored in the
244  * the main buffer `mBuffer`. mBuffer is utilized as a circular buffer.
245 
246  * Messages (which are added using `InFrameFeedMessaged()`) are not copied in the `mBuffer` but instead are
247  * enqueued in a message queue `mMessageQueue`.
248  *
249  * The data segments include a header before the data portion. The header is 2 bytes long is formated as follows
250  *
251  * Bit 0-13: Give the length of the data segment (max segment len is 2^14 = 16,384 bytes).
252  * Bit 14: Flag bit set to indicate that this segment has an associated `Message` (appended to its end).
253  * Bit 15: Flag bit set to indicate that this segment defines the start of a new frame.
254  *
255  * Bit 15 Bit 14 Bits: 0 - 13
256  * +--------------+--------------+--------------------------------------------------------+
257  * | New Frame | Has Message | Length of segment (excluding the header) |
258  * +--------------+--------------+--------------------------------------------------------+
259  *
260  * The header is encoded in big-endian (msb first) style.
261 
262  * Consider the following calls to create a frame:
263  *
264  * ncpBuffer.InFrameBegin();
265  * ncpBuffer.InFrameFeedData("Hello", 5);
266  * ncpBuffer.InFrameFeedData("There", 5);
267  * ncpBuffer.InFrameFeedMessage(*someMessage);
268  * ncpBuffer.InFrameFeedData("Bye", 3);
269  * ncpBuffer.InFrameEnd();
270  *
271  * This frame is stored as two segments:
272  *
273  * - Segment #1 contains "HelloThere" with a header of `0xC00A` which shows that this segment contains 10 data
274  * bytes, and it starts a new frame, and also must include a message from the message queue.
275  *
276  * - Segment #2 contains "Bye" with a header value of `0x0003` showing length of 3 and no appended message.
277  *
278  * +-----+-----+-----+-----+-----+-----+-----+-----+-----+-----+-----+-----+-----+-----+-----+-----+-----+
279  * | C0 | 0A | `H` | 'e' | 'l' | 'l' | 'o' | 'T' | 'h' | 'e' | 'r' | 'e' | 00 | 03 | 'B' | 'y' | 'e' |
280  * +-----+-----+-----+-----+-----+-----+-----+-----+-----+-----+-----+-----+-----+-----+-----+-----+-----+
281  * \ / \ /
282  * Segment #1 Header Segment #2 Header
283  *
284  */
